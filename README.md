# Fract-ol
## 42 School — Graphics & Fractals
### by hdruel

## Description
Fract-ol is a graphics programming project from the 42 curriculum. Its goal is to introduce computer graphics and complex number mathematics by rendering beautiful, infinitely zoomable fractal sets using the MiniLibX graphics library.

The program renders fractals in a window, allowing the user to explore them interactively through zoom, pan, and color controls.

The project includes:
    • Rendering of the Mandelbrot, Julia, Burning Ship and Tricorn sets
    • Smooth infinite zoom using mouse scroll
    • Julia set parameter variation
    • Customizable color palettes
    • Keyboard controls for navigation and settings

Project Architecture and Design Choices
## What is a Fractal?
A fractal is a geometric shape that is self-similar at every scale — zooming in reveals the same level of complexity. Fractals are generated by iterating a mathematical function on complex numbers and checking whether the result diverges or stays bounded.

The key iteration formula for the Mandelbrot set is:
z(n+1) = z(n)² + c
where z starts at 0 and c is the point being tested

For the Julia set, c is a fixed constant and z is initialized to the point being tested, producing a different fractal for each value of c.

MiniLibX was chosen as required by the project. It provides direct pixel access, which is ideal for understanding how graphics rendering works at a low level.

## Complex Numbers
Fractals are computed in the complex plane. Each pixel on screen maps to a complex number c = real + i*imag. The iteration formula is applied repeatedly, and the number of iterations before divergence determines the pixel's color.

This approach means:
    • The x-axis of the screen maps to the real part of c
    • The y-axis maps to the imaginary part of c
    • Zoom is achieved by narrowing the range of complex values mapped to the screen

## Rendering Performance
Each pixel requires many iterations to determine its color. To keep the program responsive:
    • The maximum iteration count is capped (adjustable by the user)
    • Pixels that never diverge (inside the set) reach the cap and are colored black
    • Color smoothing techniques reduce visible banding between iteration counts

## Color Palettes
Colors are mapped to the number of iterations it took for a point to escape. By cycling through color gradients based on this count, visually striking patterns emerge at the fractal boundaries. The palette can be changed interactively.

## Instructions
Prerequisites
    • GCC / Clang compiler
    • make
    • MiniLibX (included or provided by 42)
    • X11 libraries (Linux) or AppKit (macOS)

Compilation
make

## Launch the program
Usage:
./fractol <fractal_name> [options]

Available fractals:
    • mandelbrot — The classic Mandelbrot set
    • burnig ship — The classic Mandelbrot set
    • tricorn — The classic Mandelbrot set
    • julia [real] [imag] — Julia set with optional complex parameter c

## Examples
Examples
# Render the Mandelbrot set
./fractol mandelbrot
 
# Render a Julia set with default parameter
./fractol julia
 
# Render a Julia set with custom c = -0.7 + 0.27i
./fractol julia -0.7 0.27
 
# Render the Burning Ship set
./fractol burningship
 
# Render the Tricorn set
./fractol tricorn

## Controls
Once the window is open:
    • Scroll up/down — Zoom in / zoom out at mouse position
    • Arrow keys — Pan the view
    • left click — Update Julia set parameter in real time (Julia mode)
    • Space — Cycle through color palettes
    • ESC — Close the program

AI was not used to generate the project automatically. All C source files, the MiniLibX event handling, the fractal rendering logic, and the Makefile were implemented, tested, and debugged manually.
